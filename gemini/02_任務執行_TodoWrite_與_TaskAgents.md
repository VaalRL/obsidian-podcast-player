# 02｜任務執行：TodoWrite 與 Task Agents

> 適用於 Obsidian Podcast Player 插件開發

## 何時使用
- 任務 **≥3 步**或需多人/多代理並行
- 操作 **>30 秒** 或需背景持續（context 切換不會中斷）
- 修改多個模組（如 podcast + player + ui）

## 作法
1. **TodoWrite**：把任務拆成清單（可平行），為每項標示輸入/輸出、完成定義、風險、Rollback。
2. **啟動 Task Agents**：長任務交由 Task Agent 執行；Bash 在 context 切換會中斷。
3. **檢查點策略**：重要步驟後做 Git checkpoint；必要時建立備份。
4. **驗收**：以測試與成功指標驗證每一子任務。

## 專案特殊考量
- **模組間依賴**：修改播放器功能時，確認不會影響播放清單與佇列
- **UI 更新**：修改資料模型後，確認 UI 顯示正確更新
- **設定相容性**：新增設定時，確保舊版本相容
- **音訊播放**：修改播放邏輯時，確認進度追蹤與狀態同步
- **Feed 同步**：修改 Feed 解析時，確認與現有訂閱相容

## 產出
- Todo 清單（含優先順序、阻塞項）
- 自動化測試與驗收報告
- 記錄關鍵設計決策與進度
